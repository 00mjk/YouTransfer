YouTransfer

The simple but elegant file-sharing solution

Learn more

YouTransfer is a simple but elegant self-hosted file transfer & sharing solution. It is an alternative to paid services like Dropbox and WeTransfer by offering similar features but without limitations, price plans and a lengthy privacy policy. You remain in control of your files.

Created to be installed behind the firewall on private servers, YouTransfer aims to empower organisations and individuals that wish to combine easy-to-use file transfer tooling with security and control.

Quick Start using Docker

If you wish to run YouTransfer in your own environment without any modifications, the Docker image is the quickest and easiest approach. All you have to do is install docker and run:

docker pull remie/youtransfer:stable

Once the image has been downloaded, start the application with the following command: 

docker run -d 
-v [path_to_upload_folder]:/opt/youtransfer/uploads 
-v [path_to_config_folder]:/opt/youtransfer/config 
-p 80:5000 
remie/youtransfer:stable

You can now connect to YouTransfer by browsing to http://[docker_host_ip]/
For more information on Docker deployment, please read the Docker installation instructions.

Self-hosted using NodeJS

If you wish to install YouTransfer directly from source, you will need to install NodeJS. Download the latest release zip file from GitHub or clone the repository from https://github.com/remie/YouTransfer.git

To install YouTransfer, run npm install in the application directory.
After installation, start YouTransfer with PORT=[port] npm start.

You can now connect to YouTransfer by browsing to http://localhost:[port]/
For more information on hosting YouTransfer please read the hosting instructions.

Contribute

YouTransfer is an Open Source project hosted on GitHub. If you wish to contribute (hurray!) to YouTransfer, make sure to:

  1. Fork the project on GitHub and clone it
  2. Get your local development environment up and running with npm install, npm run dev and npm run app
  3. Work on one of the existing issues, or on your own feature.
  4. Discuss your work on Gitter
  5. Test your work with npm test
  6. Make sure to upmerge the latest version of master into your fork / branch.
  7. Send a pull request when you're done! Make sure to update / include unit tests (we aim to have 100% code coverage) and check JSHint (we're very proud of our 4.0 GPA at Code Climate). Pull requests will be automatically build by Travis CI.

You're pull requests will be reviewed and if approved it will be merged. Make sure to be available to discuss your pull request and make changes if required.
